# 配置指南

## 配置文件

| 文件 | 用途 |
| --- | --- |
| `config/providers.yaml` | provider 连接信息，如 base_url 和 api_key_env |
| `config/models.yaml` | 可提交的模型 role 模板和默认值 |
| `config/models.local.yaml` | 用户本地模型选择覆盖，默认被 `.gitignore` 忽略 |
| `config/agent.yaml` | 运行参数和 fallback system prompt |
| `config/agents.yaml` | Agent 身份、能力、边界和风格 |
| `config/tools.yaml` | 工具和 ToolPack 启用配置 |
| `config/voice.yaml` | dummy 或真实语音配置 |
| `config/mcp.yaml` | MCP server 占位配置，默认关闭；V0.1 不向 LLM 注册 MCP tools/call |

## 模型配置闭环

远程 profile 在 `models list` 前必须先设置 API Key 环境变量；本地 profile 要先启动对应 OpenAI-compatible 服务。

```powershell
mini-agent init --profile <profile>
$env:<API_KEY_ENV>="你的 API Key"  # 仅远程 profile 需要
```

```powershell
mini-agent models list --profile <profile>
mini-agent models use --profile <profile> --model "<模型ID>"
mini-agent models check --profile <profile>
mini-agent models doctor --profile <profile>
```

`models list` 只读 `/models`，不写配置。`models use` 只写 `config/models.local.yaml -> profiles.<profile>.roles.main.model`，不改 provider、base_url、API Key、state 或被 git 跟踪的 `config/models.yaml`。

## 查看配置

```powershell
mini-agent config where --profile <profile> --role main
mini-agent config show --profile <profile>
mini-agent config show --profile <profile> --show-model
mini-agent config show --profile <profile> --show-examples
```

默认 `config show` 只显示 `<已配置>` 或 `<未配置>`，避免把用户模型名带到日志里。

## state 安全

`mini-agent use` 写入 `.mini-agent/state.json`，只保存：

- `default_profile`
- `default_agent`
- `default_mode`
- `config_dir`

禁止保存模型名、API Key、base_url、example_models 或任何密钥。
